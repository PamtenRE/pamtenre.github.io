🌿 Branching Strategy

Default branch: develop
Create new feature branches from it using: 

feature/<component-name>

Example: feature/login-ui or feature/resume-upload

🔁 Pull Request & Merge Process

  1. Open a PR from your feature/<component> branch into develop.
  2. Any teammate from your repo can review and approve the PR.
  3. After approval and checks, merge your PR.

Develop → auto-deploys to DEV environment in Azure.
Develop → Master (or Release) → PROD.

✅ Best Practices

  • Never push directly to develop or master.
  • Keep commits clean and descriptive.
  • Pull latest develop before creating new branches.
  • Test locally before opening a PR.
  • Use small, focused PRs for easier reviews.
